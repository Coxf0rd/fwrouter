# fwrouter

Публичный репозиторий домашнего шлюза на Linux с двумя частями:

- `fwrouter-stack/` — сам шлюз: `fwrouter` UI/API, `mihomo2`, systemd-юниты, host-скрипты, примеры `/etc/fwrouter/*`
- `vless-gateway/` — опциональная раздача VLESS-подписок для клиентов

Проект рассчитан на мини-ПК или другой Linux-хост, который стоит на входе в локальную сеть и управляет маршрутизацией трафика для домашних устройств.

## Что умеет

- UI с разделением на пользовательский и админский режимы
- выбор VPN-сервера вручную или через `vpn-auto`
- режимы `DIRECT`, `VPN`, `SELECTIVE`
- per-device overrides для клиентов
- отдельный режим для трафика самого шлюза (`self_mode`)
- обновление и пере-применение Re-filter списков из GitHub Releases
- локальные флаги стран в UI без зависимости от внешнего CDN
- отдельные проверки клиентского IP для `DIRECT` и `VPN`

## Какой сценарий использования

### 1. Только шлюз

Используется `fwrouter-stack/`.

Это вариант, когда:

- шлюз сам поднимает `mihomo2`
- локальные устройства идут через правила `fwrouter`
- сервер VPN для трафика через шлюз выбирается глобально в одном инстансе `mihomo`

Важно: в этой схеме все устройства, которые идут через VPN именно шлюза, используют один текущий сервер из группы `PROXY`.

### 2. Шлюз + локальная VLESS-подписка

Используются `fwrouter-stack/` и `vless-gateway/`.

Это вариант, когда:

- шлюз продолжает работать как роутер с политиками
- дополнительно поднимается подписка для внешних клиентов
- клиенты со своими приложениями получают список VLESS-нод и уже в своем приложении выбирают нужный сервер независимо от `fwrouter`

То есть:

- `fwrouter`-маршрутизация = один активный upstream на `mihomo`
- клиенты с отдельной VLESS-подпиской = каждый может выбрать свой сервер сам

## Быстрый старт

```bash
apt-get update
apt-get install -y git ansible
git clone https://github.com/Coxf0rd/fwrouter.git /opt/fwrouter
cd /opt/fwrouter
ansible-playbook -i 'localhost,' -c local ansible/playbook-base.yml
```

После установки:

1. Открой UI по адресу `http://<LAN-IP-шлюза>:9280/`
2. Во вкладке `Подписка` вставь URL своей подписки
3. Выбери режим маршрутизации и сервер

## Что требуется дополнительно

### Для `fwrouter-stack`

- Debian/Ubuntu-подобная система с `systemd`
- root/sudo
- Docker
- `dnsmasq`, `iptables`, `ipset`
- `/dev/net/tun`

Белый IP и домен не обязательны, если UI нужен только внутри локальной сети.

### Для `vless-gateway`

Понадобятся:

- белый IP или проброс портов до шлюза
- домен
- REALITY ключи или TLS-конфигурация, в зависимости от схемы

Эти секреты в репозиторий не входят.

## Что не хранится в репозитории

- реальные URL подписок
- приватные ключи, сертификаты, `*.key`, `*.pem`
- REALITY private key
- реальные токены и секреты из `.env`

## Примечания по текущей реализации

- IP в UI проверяется на стороне браузера клиента, а не на стороне сервера API
- для флагов используются локальные SVG из `fwrouter-stack/fwrouter/api/app/static/flags/`
- обновление `SELECTIVE` ускорено: `fwrouter-apply` не пересобирает тяжелые dnsmasq-правила без необходимости
